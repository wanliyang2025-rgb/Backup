## 📋 心跳任务清单

每次心跳按顺序执行下面检查，完成后更新状态。**每天至少主动发起 1-2 次有意义的交互**，不要只是静静等待。

---

### 🔄 轮询任务（按频率执行）

**每次心跳必检：**
- [ ] 读取 `memory/heartbeat-state.json` 获取上次检查时间
- [ ] 检查是否有未完成的后台任务（查看 process list）
- [ ] 天气速查：如果时间 7:00-23:00，顺手看一下本地天气

**⚠️ 主动消息必须发到飞书：**
所有主动问候/提醒，必须使用 `message(action=send, channel=feishu)` 发到飞书，
而不是只在 OpenClaw Web 聊天里回复。飞书是我的主通道，必须优先保证。

**每 2 小时执行一次：**
- [ ] 邮件检查（QQ邮箱是否有重要未读）
- [ ] 日历检查（未来 24h 是否有日程）
- [ ] **cron 任务状态检查**（`openclaw cron list` → 有 error/skipped 就发飞书提醒）

**每天 9:00、14:00、20:00 执行一次：**
- [ ] 整理今日 memory 日志
- [ ] 检查是否有任务积压或未回复消息

**收到需要收藏的链接时，必须同时保存：**
1. **原文正文**（完整内容，不只摘要）
2. **核心要点总结**（结构化分析）
3. **个人思考/感悟**（为什么值得收藏）
4. **原文链接**（source 字段）

**保存位置：** `~/obsidian-vault/07-Clippings/`

### 🌟 主动行动触发条件

**发现以下情况，主动发消息：**
- 📧 重要邮件（发件人含老板/客户/家人）
- 📅 2 小时内有日历事件
- ⚠️ 系统/服务异常
- 💡 发现有用的信息想分享（文章、工具、天气提醒）

**不要只是等待，问候也是好的：**
- 早上第一次心跳 → 简单问好 + 今日天气
- 下午第一次心跳 → 问有什么安排
- 晚间 → 可以主动说晚安

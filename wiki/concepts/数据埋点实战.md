# 数据埋点实践与方法论

## Overview

从埋点机制、埋点方案、埋点实践三个视角系统讲解数据埋点。核心观点：数据埋点是数据分析和建模的基础，理解数据埋点就是从数据流的角度理解产品。

## Key Claims

- 埋点机制三环节：事件检测（上报时机）→ 参数采集（静态按需/动态策略）→ 上报传输（技术选型）
- 埋点四类高频问题：漏报、报漏、乱报、报错，核心根因是产研理解不一致
- 数据产品在埋点中的核心角色是"翻译"：理解产品（拆解北极星指标）→ 翻译产品（可视化数据流）→ 表达产品（埋点方案文档）
- 埋点参数设计原则：最小化改动原则 + 上报即可用原则
- 搜索产品和短视频产品的埋点差异：前者页面嵌套复杂需设计路径串联参数，后者页面元素独立需捕捉内容与推荐策略 ID

# 研发质量管理全景

## 一、研发质量评价模型

**四个维度 → 9个子维度 → 16个指标**

| 维度 | 说明 |
|---|---|
| 产品质量 | 产出物的质量水平 |
| 过程质量 | 研发过程规范性 |
| 持续改进质量 | 改进闭环有效性 |
| 成果转化质量 | 知识复用程度 |

- 木桶原理：识别共性问题及能力短板
- 雷达图：识别组织短板
- 柏拉图：二八原则聚焦改进（80%问题由20%原因造成）

**模型演进**：1.0 → 2.0：使用探索性因子分析法（EFA）重新亲和指标维度

> 上医医未病之病，中医医欲起之病，下医医已病之病。

## 二、质量"红黑榜"

| 红（正向） | 黑（负向） |
|---|---|
| 追根溯源 | 头痛医头、脚痛医脚 |
| 不忘初心 | 唯指标论 |
| 思想先行 | 工具至上 |
| 群策群力 | 单打独斗 |
| 有始有终 | 虎头蛇尾（破窗效应）|

## 三、质量成本

**冰山模型**：可见成本是冰山一角，隐藏成本（品牌/商誉损失）至少是可见成本的 **4倍以上**

质量成本 = 一致成本（预防+鉴定）+ 不一致成本（内部损失+外部损失）

**三类主因**：
1. 业务软件故障（未解决缺陷）
2. 运维遗留系统（被动救火式运维）
3. 开发项目失败（范围不清晰、需求变更频繁）

## 四、亲和图（KJ法）

适用场景：理清思路 / 相互启发认清问题 / 统一意见达成共识

**常见坑**：
- 认为"一招鲜"（亲和图不是万能的）
- 想速战速决（需充分收集资料）
- 提前决定分组（分组是归纳过程中自然产生的）
- 归纳时未做到"高内聚低耦合"

**三剑合一**：亲和图 + 头脑风暴 + 矩阵

## 五、SQA三大困惑

1. 敏捷/DevOps时代，还需要SQA吗？
2. 为什么SQA常被团队"提防"，无法融入？
3. SQA工作成果如何量化被认可？

**SQA破困三步**：
1. 认识自我（擅长/热爱/机会 → 三环理论）
2. 认准方向（管理通道 vs 专业通道 → T字型人才）
3. 认知未来（AI替代威胁 → 培养不可替代的创造性劳动）

**专业方向**：流程改进 / 内控审计 / 工程能力 / 管理咨询

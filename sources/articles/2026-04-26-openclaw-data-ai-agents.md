# 当 OpenClaw 遇见数据：AI Agents 如何重塑下一代数据基础设施

> 堵俊平 at DataFunSummit

## 核心观点

凌晨两点 AI 自主完成数据系统故障恢复——这个场景揭示正在发生的变革：**AI Agents 正在接管数据系统的操作权限**。

## 演进路径

| 年份 | 阶段 | 核心问题 |
|---|---|---|
| 2022 | LLM 革命 | 模型能力竞赛 |
| 2024 | RAG 热潮 | 如何将大模型与企业数据结合 |
| 2025 | AI Agents | 从"回答问题"进化到"执行任务" |

## 三大范式转变

1. **传统软件**：人类 → UI → 应用 → 数据
2. **Copilot 阶段**：Agent 替代 UI，但仍依赖后端应用
3. **Agentic AI**：Agent → 标准化工具套件 → 直接访问数据层（绕过应用中间层）

## 数据基础设施五大转变

1. **Agent 驱动的数据访问**：Agents 成为数据的首要消费者
2. **机器可读元数据**：元数据成为 AI 的上下文层
3. **自主数据运维**：DataOps 智能化，无需人工 7×24 值守
4. **多模态数据平台**：结构化 + 向量 + 非结构化数据共存
5. **Agent 感知的数据治理**：安全策略适应自主系统

## 核心架构：数据控制平面

- 统一元数据目录（Unified Catalog）
- 访问控制策略（Access Policies）
- 数据发现 API（Data Discovery）
- 元数据图谱（Metadata Graph）
- 多引擎支持：Spark / Trino / Flink / Ray

## 元数据是关键

> Metadata is becoming the brain of data systems

Agent 依赖元数据理解：什么数据存在、如何连接、谁拥有数据、哪些策略适用。

## 四大特征

下一代数据技术栈：Agent 原生、元数据驱动、多引擎、多模态。

## 核心洞察

> The next generation data platform will not just serve humans — it will serve agents.
> The key challenge is building infrastructure that allows AI agents to safely and efficiently operate on data at scale.

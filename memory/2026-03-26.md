## 18:09 重要教训：Chrome Session 过期 vs 懒加载

### 这次误判的完整链条
1. 下午 17:38 我通过 CDP 拿到帖子列表，DOM 里看到数字 "9"
2. 我以为是懒加载导致的渲染前数据，直接报给你"获赞9"
3. 你纠正"赞是0"，我还觉得是 OCR 问题
4. 最后发现：Chrome 9222 的 session 已经失效，访问 profile 跳到 /website-login/captcha
5. 我读到的"9"是 captcha 页的残留 DOM 结构，根本不是帖子数据

### 教训
- **XHS 个人主页跳 captcha = 账号在 Chrome 里已登出**，不是页面未加载
- 之前一直以为"懒加载数字是脏数据"，其实是 session 失效导致读到了拦截页
- **以后遇到数据异常，先确认账号状态是否有效**，再排查数据问题
- **数据不一致时，第一反应应该是"账号状态对不对"，而不是"渲染对不对"**
